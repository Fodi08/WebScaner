# Lite Web Vulnerability Scanner

Образовательный инструмент на Python для базовой оценки безопасности веб-приложений. Разработан для изучения принципов HTTP-безопасности, детектирования уязвимостей и генерации отчётности.

## Возможности

- Проверка заголовков безопасности: обнаружение отсутствующих HSTS, CSP, X-Frame-Options, X-Content-Type-Options
- Поиск открытых чувствительных файлов: сканирование на наличие .env, .git/config, backup.sql, wp-config.php и других критических файлов
- Генерация отчёта в формате Markdown: экспорт результатов в структурированный файл report.md
- Обработка ошибок: корректная обработка таймаутов и ошибок подключения без аварийного завершения

## Описание модулей
core/session.py
Настраивает HTTP-сессию для сканера:
* Установка заголовка User-Agent для имитации обычного браузера
* Отключение предупреждений о самоподписанных SSL-сертификатах
* Настройка таймаутов и параметров повторных попыток

checks/headers.py
Проверяет наличие критически важных заголовков безопасности:
* Strict-Transport-Security (HSTS) — защита от downgrade-атак
* X-Frame-Options — защита от clickjacking
* X-Content-Type-Options — защита от MIME-sniffing
* Content-Security-Policy — защита от XSS-инъекций

checks/exposed_files.py
Сканирует корень сайта на наличие чувствительных файлов:
* Файлы конфигурации (.env, config.php, wp-config.php)
* Резервные копии баз данных (backup.sql, database.sql)
* Системные файлы (.git/config, phpinfo.php)
* Файлы с метаданными (robots.txt, sitemap.xml)

core/reporter.py
Генерирует отчёт в формате Markdown:
* Сводная таблица по уровням опасности
* Детализированный список найденных проблем
* Автоматическое добавление даты и целевого хоста
* Дисклеймер об образовательном назначении инструмента

## Расширение функционала
Проект имеет модульную архитектуру, что упрощает добавление новых проверок:
1. Создайте новый файл в папке checks/, например checks/new_check.py
2. Реализуйте функцию с сигнатурой def check_name(session, url):, возвращающую список словарей с результатами
3. Импортируйте функцию в main.py и добавьте вызов в основной цикл
4. При необходимости обновите reporter.py для отображения новых типов находок

Примеры возможных расширений:
* Проверка на уязвимости инъекций (SQLi, XSS)
* Определение технологий (CMS, веб-сервер, фреймворки)
* Сканирование поддоменов
* Интеграция с внешними API (VirusTotal, Shodan)

Ограничения
* Инструмент выполняет только пассивное сканирование (без эксплойтов)
* Проверки ограничены корневым путем сайта (без глубокого обхода)
* Отсутствует поддержка аутентификации (сканирование только публичных ресурсов)
* Результаты требуют ручной верификации перед принятием решений
